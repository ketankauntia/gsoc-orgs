# AGENTS.md

## Cursor Cloud specific instructions

### Overview

GSoC Organizations Guide — a Next.js 16 (App Router) web app with React 19, TypeScript, Tailwind CSS 4, Prisma + MongoDB, and Recharts. See `README.md` for full tech stack and getting started guide.

### Services

| Service | Purpose | Start command |
|---|---|---|
| **Next.js dev server** | Main application | `pnpm dev` (http://localhost:3000) |
| **MongoDB** | Database (Prisma ORM) | `mongod --dbpath /data/db --logpath /data/db/mongod.log --bind_ip 127.0.0.1 --port 27017 &` |

MongoDB must be running before the dev server starts. The connection string is configured via `DATABASE_URL` in `.env.local`.

### Common commands

Standard commands are in `package.json` scripts — key ones:

- **Lint:** `pnpm lint`
- **Type-check:** `pnpm type-check`
- **Validate (lint + type-check + build):** `pnpm validate`
- **Dev server:** `pnpm dev`
- **Build:** `pnpm build`

### Non-obvious caveats

- **Pre-commit hooks run a full build.** Husky's `pre-commit` hook runs lint-staged, `tsc --noEmit`, and `pnpm build`. Use `git commit --no-verify` to skip during iterative development, but ensure `pnpm validate` passes before the final push.
- **Branch protection in hooks.** The pre-commit hook blocks commits to `main`/`master`. Always work on feature branches.
- **`pnpm install` triggers `prisma generate` automatically** via the `postinstall` script. No need to run it separately.
- **Some pages use precomputed JSON data** (in `/data/` directory or generated by scripts in `/scripts/`), not just live MongoDB queries. The yearly and tech-stack pages render data even with an empty local database.
- **The `/api/v1/*` endpoints query MongoDB directly** and will return empty results if the local database has no data. This is expected for local development without a populated database.
- **`pnpm approve-builds` is interactive** and should not be run in non-interactive environments. The warning about ignored build scripts from pnpm can be safely ignored — `prisma generate` runs via the `postinstall` script.
- **Commitlint enforces conventional commits.** Messages must follow format: `<type>: <description>` (e.g., `feat: add search filter`).
